DESCRIPCIÓN DEL PROYECTO
================================================================================

Agenda Virtual EIWA es una plataforma web colaborativa diseñada para gestionar
tareas, grupos de estudio y calendarios académicos. Permite a estudiantes y
profesores organizar actividades educativas, compartir tareas, y colaborar en
grupos con diferentes niveles de permisos y roles.

La aplicación está optimizada para dispositivos móviles y de escritorio, con
soporte para modo oscuro, notificaciones en tiempo real, y una experiencia de
usuario moderna y accesible.

APLICACIONES DJANGO (APPS)
================================================================================

1. ACCOUNTS (Cuentas de Usuario)
---------------------------------
Propósito: Gestión de usuarios, autenticación y perfiles

Modelos:
- User: Usuario personalizado extendiendo AbstractUser
  * Campos: nombre, apellido, email (único)
  * USERNAME_FIELD: email
  * Autenticación por correo electrónico

Funcionalidades:
- Registro de usuarios
- Login/Logout
- Gestión de perfiles
- Cambio de contraseña
- Eliminación de cuenta

Vistas principales:
- register: Registro de nuevos usuarios
- login_view: Inicio de sesión
- logout_view: Cierre de sesión


2. GROUPS (Grupos/Cursos)
--------------------------
Propósito: Gestión de grupos de estudio con roles y permisos

Modelos:
- Group: Grupo/Curso principal
  * Información: nombre, descripción, max_members
  * Código de invitación único (8 caracteres)
  * Tipos de ingreso: libre o con aprobación
  * Permisos configurables para tareas y materias
  
- GroupMember: Miembros del grupo
  * Roles: líder, co-líder, miembro
  * Sistema de advertencias
  * Fecha de ingreso
  
- JoinRequest: Solicitudes de ingreso
  * Estados: pendiente, aprobada, rechazada
  * Mensaje opcional
  * Revisión por líder
  
- BannedUser: Usuarios baneados
  * Razón del baneo
  * Usuario que aplicó el baneo
  
- GroupActivity: Historial de actividades
  * 16 tipos de actividades diferentes
  * Metadata en JSON
  * Registro completo de cambios

Funcionalidades:
- Crear grupos con código de invitación
- Unirse a grupos (libre o con aprobación)
- Sistema de roles (líder, co-líder, miembro)
- Gestión de miembros (expulsar, banear, advertir)
- Configuración de permisos granulares
- Historial de actividades del grupo
- Regeneración de códigos de invitación

Permisos configurables:
- Creación de tareas: todos, solo líder, con aprobación
- Edición de tareas: todos, solo líder, con aprobación
- Eliminación de tareas: líder, líder+creador, todos
- Reversión de cambios: líder, líder+creador
- Gestión de materias: líder, sugerir, todos


3. TASKS (Tareas)
-----------------
Propósito: Gestión de tareas académicas

Modelos:
- Task: Tarea principal
  * Información: título, descripción, páginas
  * Fechas: asignada, entrega
  * Estado: pendiente, completada
  * Prioridad: baja, media, alta
  * Relación con grupo y materia
  
- TaskRequest: Solicitud de creación de tarea
  * Para grupos con aprobación requerida
  * Estados: pendiente, aprobada, rechazada
  
- TaskEditRequest: Solicitud de edición
  * Cambios propuestos en JSON
  * Requiere aprobación según permisos
  
- TaskCompletion: Estado por usuario
  * Cada estudiante marca su progreso
  * Fecha de completado individual

Funcionalidades:
- Crear tareas con fechas y prioridades
- Editar y eliminar tareas (según permisos)
- Marcar tareas como completadas (individual)
- Sistema de solicitudes para creación/edición
- Detección de tareas vencidas
- Filtrado por materia, estado, prioridad


4. SUBJECTS (Materias)
----------------------
Propósito: Gestión de materias/asignaturas

Modelos:
- Subject: Materia académica
  * Nombre y descripción
  * Color identificador
  * Relación con grupo
  * Creador y fecha de creación

Funcionalidades:
- Crear materias por grupo
- Asignar colores para identificación visual
- Asociar tareas a materias
- Permisos según configuración del grupo


5. CALENDAR_APP (Calendario)
----------------------------
Propósito: Visualización de tareas en calendario

Funcionalidades:
- Vista mensual de tareas
- Filtrado por grupo y materia
- Indicadores visuales de prioridad
- Navegación entre meses
- Integración con tareas y fechas de entrega


6. NOTIFICATIONS (Notificaciones)
---------------------------------
Propósito: Sistema de notificaciones en tiempo real

Modelos:
- Notification: Notificación
  * Tipos: tarea, grupo, sistema
  * Estados: leída/no leída
  * Enlace a recurso relacionado
  * Fecha de creación

Funcionalidades:
- Notificaciones en tiempo real
- Notificaciones push (navegador)
- Centro de notificaciones
- Marcar como leídas
- Configuración de preferencias
- Badges de contador


7. TRACKING (Seguimiento)
-------------------------
Propósito: Seguimiento de actividades y estadísticas

Funcionalidades:
- Registro de actividades del usuario
- Estadísticas de tareas completadas
- Historial de cambios
- Análisis de productividad

Componentes:
